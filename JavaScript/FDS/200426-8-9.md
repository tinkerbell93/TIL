## 제어문

문은 일반적으로 위에서 아래로 실행되지만 모두 위에서 아래로 실행되지는 않는다. 코드의 흐름을 바꿀 수 있는데 이것을 제어문으로 한다. 하지만 코드의 흐름을 왜곡시키면 가독성이 떨어진다. 제어문은 가독성에 좋지 않기 때문에 굉장히 심플하게 사용하는 것이 좋다.

어떤 조건에 의해 실행을 제어 하는 것이다.

#### 2.1 if...else 문

조건식은 표현식에 포함된다.

```javascript
if (조건식) {
  // 조건식이 참이면 이 코드 블록이 실행된다.
} else {
  // 조건식이 거짓이면 이 코드 블록이 실행된다.
}

// 위에 예제로는 경우의 수가 2개일때 사용 (true, false)

if (조건식1) {
  // 조건식1이 참이면 이 코드 블록이 실행된다.
} else if (조건식2) {
  // 조건식2이 참이면 이 코드 블록이 실행된다.
} else {
  // 조건식1, 조건식2 모두 거짓이면 이 코드 블록이 실행된다.
}

// 위에 예제로는 경의 수가 3개일때 사용
```

`else`는 옵션이다.  `if`없음 조건문 자체가 아니니까 `if`는 필수다. `else...if`는 조건에 따라 여러번 사용 가능하다. 

코드 블록 내의 문이 하나뿐이라면 중괄호`{}`를 생략할 수 있다.

```js
var num = 2;
var kind;

if (num > 0)      kind = '양수';
else if (num < 0) kind = '음수';
else              kind = '영';

console.log(kind); // 양수
```

대부분의 if…else 문은 삼항 조건 연산자 삼항 조건 연산자로 바꿔 쓸 수 있다.

```js
var x = 2;

// 0은 false로 취급된다.
var result = x % 2 ? '홀수' : '짝수';
console.log(result); // 짝수
```

> 조건이 3개일때 **삼항 조건 연산자**로 표현할 수 있다.
>
> ```javascript
> var num = 2 ;
> var kind;
> 
> kind = num > 0 ? '양수' : (num < 0 ? '음수' : '0');
> ```

- 삼항조건 연산자: 표현식인 문, 변수에 할당 가능

  ```js
  var x = 2;
  var result = x % 2 ? '홀수' : '짝수';
  ```

- if..elas문: 표현식이 아닌문, 변수에 할당 불가능

  ```js
  var x = 2;
  var result = if(x % 2){
    '홀수'
  } else {
    '짝수'
  } // Uncaught SyntaxError: Unexpected token 'if'
  ```

  

#### 2.2 switch 문

주어진 표현식을 평가하여 그 값과 일치하는 표현식을 갖는 case 문으로 실행 순서를 이동시킨다.

문법이 상당히 까다롭다. 

```javascript
switch (표현식) {
    // 표현식 : true, false로 표현되지 않아도 된다.
  case 표현식1:
    switch 문의 표현식과 표현식1이 일치하면 실행될 문;
    break;
  case 표현식2:
    switch 문의 표현식과 표현식2가 일치하면 실행될 문;
    break;
  default:
    switch 문의 표현식과 일치하는 표현식을 갖는 case 문이 없을 때 실행될 문;
}
```

`break`문은 switch문이나 반복문의 코드블럭에서 탈출(**폴스루fall through**)시킨다. 그러므로 `break`문을 꼭 사용해야 한다.

`break`문을 사용하면 falsely로 출력된다.



[ `if..else`문과 `switch`문 ]

| if..else                                  | switch                                    |
| ----------------------------------------- | ----------------------------------------- |
| 불리언 값으로 평가                        | 문자열, 숫자 값 등으로 평가               |
| 논리적 참, 거짓으로 실행할 코드 블록 결정 | 다양한 상황에 따라 실행 할 코드 블록 결정 |



### 3. 반복문

주어진 조건식의 평가 결과가 참인 경우 코드 블록을 실행한다. 그 후 조건식을 다시 검사하여 여전히 참인 경우 코드 블록을 다시 실행하며 거짓일 때까지 반복한다.

#### 3.1 for 문

```javascript
for (변수 선언문 또는 할당문; 조건식; 증감식) {
  조건식이 참인 경우 반복 실행될 문;
}
```

for의 조건식은 옵션이기때문에 생략할 수 있지만 조건이 생략 되면 무한 반복이 된다.

```javascript
// 무한루프
for (;;) { ... }
```

1중 for문은 선형(일차원), 2중 for문은 행열(2차원)을 사용할때, 3중 for문은 3차원. 그러므로 4중 for문은 4차원이며 4중 for문은 사람이 계산하기 힘들다.



#### 3.2 while 문

```javascript
while (조건식) {
  조건식이 참인 경우 반복 실행될 문;
  증감식;
}
```

while 문은 몇 번 반복 실행할지 모르지만 목표 값이 나올때까지 알아볼때 사용한다. 

```js
var count = 0;

// count가 3보다 작을 때까지 코드 블록을 계속 반복 실행한다.
while (count < 3) {
  console.log(count); // 0 1 2
  count++;
}
```

```js
var count = 0;

// 무한루프
while (true) {
  console.log(count);
  count++;
  // count가 3이면 코드 블록을 탈출한다.
  if (count === 3) break;
} // 0 1 2
```



#### 3.3 do...while 문

do...while문은 코드 블록을 먼저 평가하는 것이 아닌 실행을 하고 조건식을 평가한 후 반복할지 안할지 결정한다. 즉 코드 블록은 무조건 한번 이상 실행된다.

### 4. break 문

코드 블록을 탈출하는 것이 아니라 레이블 문, 반복문(for, for…in, for…of, while, do…while) 또는 switch 문의 코드 블록을 탈출한다. if문에서는 빠져나갈 일이 없기 때문에 `break`문을 사용하면 에러가 난다. switch, for, while문에서 사용할 수 있다.

레이블 문(Label statement)이란 식별자가 붙은 문을 말한다.(문에 이름을 붙인다. = switch문) 레이블 문은 프로그램의 실행 순서를 제어하기 위해 사용한다. 

```js
// foo라는 레이블 식별자가 붙은 레이블 문
foo: console.log('foo');
```

```js
// foo라는 식별자가 붙은 레이블 블록문
foo: {
  console.log(1);
  break foo; // foo 레이블 블록문을 탈출한다.
  console.log(2);
}

console.log('Done!');
// 1 'Done!'
```



중첩된 for문의 내부 for문에서 break 문을 실행하면 for을 탈출하여 외부 for문으로 진입하지만 내부 for문이 아닌 외부 for문을 탈출하려면 레이블 문을 사용한다.

```javascript
outer: for (var i = 0; i < 3; i++) {
  for (var j = 0; j < 3; j++) {
    // i + j === 3이면 outer라는 식별자가 붙은 레이블 for 문을 탈출한다.
    if (i + j === 3) break outer;
    console.log(`inner [${i}, ${j}]`);
  }
}

console.log('Done!');

// break outer => for문을 빠져나가라.
```

#### 자료 구조 (Data Statement)

여러개의 값을 묶어서 관리하는 이유는 그 값들이 관련이 있으니 자료 구조를 만든다. 그 중 쉬운 자료 구조는 배열(Array)이다. 배열의 값(요소)들에게 접근하는 방식은 순서(index)로 접근할 수 있다. 만약 배열의 값들이 몇 개인지 모를때 어떡해야할까?

```javascript
var arr = [1, 2, 3];
// 인간은 3라는 것을 인식할 수 있지만 컴퓨터는 인식하지 못하므로 배열의 길이만큼 조건을 달아주는 length를 사용한다.
for (var i = 0; i < arr.length; i++){
  console.log(arr[i])
}
```

```javascript
// 배열로 합계를 구할 수 있을까?

var arr = [1, 2, 3];
var sum = 0;

for (var i = 0; i < arr.length; i++){
  sum +=arr[i];
}
console.log(sum);
```

for문을 돌리기 위해서  `.length`를 제공하는데 이것을 유사 배열이라고 한다.

for문은 일정한 자료 구조를 순회할때 사용한다.



### 5. continue 문

continue는 현 시점에서 중단하고 (탈출하는 것이 아닌) 다음 코드를 실행한다. 

```js
var string = 'Hello World.';
var search = 'l';
var count = 0;

// 문자열은 유사배열이므로 for 문으로 순회할 수 있다.
for (var i = 0; i < string.length; i++) {
  // 'l'이 아니면 현 지점에서 실행을 중단하고 반복문의 증감식으로 이동한다.
  if (string[i] !== search) continue;
  count++; // continue 문이 실행되면 이 문은 실행되지 않는다.
}

console.log(count); // 3

// 참고로 String.prototype.match 메서드를 사용해도 같은 동작을 한다.
const regexp = new RegExp(search, 'g');
console.log(string.match(regexp).length); // 3
```





## 타입 변환과 단축 평가



### 1. 타입 변환이란?

값의 타입이 변환하는 것이다. 타입 변환은 크게 두가지로 나뉜다. 개발자가 의도적으로 값의 타입을 변환하는 것인 **명시적 타입 변환(Explicit coercion) 또는 타입 캐스팅(Type casting)**, 개발자의 의도와는 상관 없이 표현식을 평가하여 암묵적으로 타입이 자동 변환하는 **암묵적 타입 변환(Implicit coercion) 또는 타입 강제 변환(Type coercion)**이 있다.

예를들면 숫자타입에서 문자열 타입으로 변환하거나 불리언 타입에서 숫자타입으로 변환하거나 등등..

### 4. 단축 평가

평가하여 단축한다. 논리합, 논리곱은 반드시 불린값을 생성하는 것은 아니다.

```javascript
'Cat' && 'Dog' // 'Dog'
// 'Cat'만 보면 평가할 수 없기 때문에 'Dog'까지 보고 비교하여 평가해야한다.

'Cat' || 'Dog' // 'Cat'
// 'Cat'은 true이기때문에 'Dog'까지 평가하지 않아도 된다.
```

if 문

```javascript
var done = true;
var message = '';

// 주어진 조건이 true 일 때
if (done) massage = '완료';

// 위 코드와 같을까????
if (done) {
  massage = '완료';
}

// if문과 같은 조건인 단축 평가
message = done && '완료';

```



내가 true인지 false인지 모를때, 삼항 조건 연산자를 사용한다.

```javascript
var done = true;
var message = '';

// if...else 문
if (done) message = '완료';
else      message = '미완료';
console.log(message); // 완료

// if...else 문은 삼항 조건 연산자로 대체 가능하다.
message = done ? '완료' : '미완료';
console.log(message); // 완료
```
